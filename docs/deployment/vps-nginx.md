## Build Frontend

On a build machine or the VPS:

```bash
npm install
npm run build
```

Copy the generated `dist/` contents to:

```text
/var/www/040415.xyz/html
```

## Configure Agent API

Keep model keys in the server environment, not in frontend source.

```bash
export AGENT_API_HOST=127.0.0.1
export AGENT_API_PORT=3000
export DEEPSEEK_API_KEY=replace-with-real-server-key
export DEEPSEEK_BASE_URL=https://api.deepseek.com
export DEEPSEEK_MODEL=deepseek-chat
npm run api:start
```

For a persistent process, run the same command through `systemd`, PM2, or the deployment manager already used on the VPS. The API is stateless, so it can restart safely.

## Nginx Shape

The existing Nginx site should serve static files and proxy API routes:

```nginx
server {
    server_name 040415.xyz www.040415.xyz;

    root /var/www/040415.xyz/html;
    index index.html;

    location /api/ {
        proxy_pass http://127.0.0.1:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

Keep the existing Let's Encrypt SSL directives generated by Certbot in place.

## Smoke Checks

After starting the API process:

```bash
curl https://040415.xyz/api/health
```

Expected JSON:

```json
{"ok":true,"service":"voice-canvas-agent-api","mode":"vps-proxy"}
```

If `DEEPSEEK_API_KEY` is missing, `POST /api/agent/plan` returns a typed `MISSING_API_KEY` error. The frontend should keep using local template fallback in that case.

## China Network Notes

The MVP uses a domestic-friendly server-side proxy so users do not need direct browser access to a model provider. DeepSeek is the first adapter. The API shape is OpenAI-compatible enough to support later Qwen or Zhipu adapters without changing the frontend contract.

## Future Serverless Compatibility

The API handler is stateless by design. It can later move behind a domestic API Gateway and Function Compute runtime. If that happens, rate limiting should be configured at the gateway layer instead of relying on function memory.

See [Serverless compatibility notes](serverless-compatibility.md) for the detailed migration boundary and gateway control checklist.
